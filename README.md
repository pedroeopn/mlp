# Comparative MLP Dashboard

School project for comparing three MLP implementations on the same dataset:

- Custom implementation based on `src/core/mlp.py`
- Scikit-Learn `MLPClassifier`
- Weka `MultilayerPerceptron`

The app is built with CustomTkinter and shows one card per engine plus a chart for accuracy and training time.

## Requirements

- Python 3.10+
- `pip`
- Java if you want to use Weka
- Optional: `bin/weka.jar`

## Install

```bash
python3 -m pip install customtkinter matplotlib numpy scikit-learn
```

If you want Weka in the comparison:

1. Download a Weka 3.8+ jar
2. Create `bin/` if needed
3. Place the jar at `bin/weka.jar`
4. Make sure `java` is available in `PATH`

## Run

From the project root:

```bash
python3 -m src.main
```

The app starts with the default dataset at `src/data/dados.data-numeric`.

## What The UI Controls Mean

- `Learning Rate`, `Hidden Neurons`, and `Epochs` apply to Custom, Scikit-Learn, and Weka
- `Hidden Activation` is valid for Custom and Scikit-Learn
- `Output Activation` is valid for Custom
- Weka ignores both activation selectors in this integration

All three engines use the same split generated by the app:

- test size: `20%`
- random seed: `42`

## How The Custom Engine Works

`src/core/mlp.py` is kept as the professor's implementation. The app wrapper calls `mlp_treino(...)` and `mlp_teste(...)` directly and only packages the result for the UI.

## Dataset Format

The dataset must be numeric and whitespace-separated, with the class in the last column:

```text
feature_1 feature_2 ... feature_n class
```

## Notes

- Weka is optional; if `bin/weka.jar` is missing, only the Weka card fails
- The chart compares `Accuracy` and `Time`
- The app supports `Start` and `Cancel` for each comparison run
